# Goal

## User Problem

New users may keep books on a computer or in an OPDS catalog, but onboarding currently exposes only the system Files picker.

## Target User

Apple ecosystem readers arriving with a local file, a computer-side collection, or an existing OPDS feed.

## Desired Behavior Change

Users choose the source they already use, import one book, and continue to Watch target selection and Reader without leaving onboarding.

## Success Criteria

- Files, Wi-Fi transfer, and OPDS can each produce the selected onboarding Publication.
- Each successful source continues to the same control-target step on iPhone and Reader step on iPad.
- Local-network permission is not requested until Wi-Fi transfer is selected.

## Non-Goals

- Redesigning Wi-Fi transfer or OPDS browsing.
- Adding cloud accounts or book synchronization.
- Adding analytics SDKs or funnel tracking.
