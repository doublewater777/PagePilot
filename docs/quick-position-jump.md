# 快速位置跳转功能规格

## 目标

让用户长按 Reader 底部的阅读位置指示器，通过一次水平拖动快速预览并跳转到出版物中的任意 Reading Position。

本功能统一使用 Readium 生成的 Reading Position。对于可重排 EPUB，它不是当前字号和屏幕尺寸下的实际排版页；产品文案、代码和测试不得将其定义为屏幕页码。

## 非目标

- 不计算 EPUB 当前排版下的屏幕页数。
- 不替代有声书的时间轴。
- 第一版不增加 VoiceOver 操作方式。
- 不增加功能教学、首次使用提示或永久可交互样式。
- 不增加触觉反馈设置，也不复用 Apple Watch 的“翻页时振动”设置。

## 支持范围

| 维度 | 支持范围 |
| --- | --- |
| 设备 | iPhone、iPad |
| 出版物 | EPUB、CBZ、PDF |
| 阅读模式 | 分页、滚动 |
| 输入 | 第一版仅支持直接触摸 |

设备没有可用触觉硬件时，快速位置跳转仍然可用，触觉反馈自然降级为无反馈。

## 入口与激活

- 保持现有底部位置指示器的视觉样式和位置。
- 指示器的可触控区域不得小于 `44 × 44 pt`。
- 长按满 `0.5 秒`后进入拖动状态。
- Reading Position 列表尚未准备好或为空时，长按无反应，不显示加载或错误提示。
- 未达到长按时长的短按继续执行现有行为：切换 Reader 导航栏。

## 拖动与目标映射

长按成功时记录：

- 当前 Reading Position `p0`；
- Reading Position 总数 `N`；
- 手指激活横坐标 `x0`；
- 当前可用水平交互范围的左右边界 `left`、`right`。

手指没有水平移动时，目标必须保持为 `p0`。左右两侧分别映射剩余范围，使一次手势可以抵达全书任意位置：

```text
x <= x0: target = 1 + round((p0 - 1) × (x - left) / (x0 - left))
x >= x0: target = p0 + round((N - p0) × (x - x0) / (right - x0))
```

目标取最近的有效 Reading Position，并始终限制在 `1...N`。实现需要处理激活点恰好落在水平边界的情况，避免除以零。

拖动期间只改变预览目标，正文、已保存的 Reading Progress 和 Watch 进度均保持不变。

## 位置切换器

长按成功后，在底部指示器上方显示紧凑的悬浮数字胶囊，不显示传统滑杆。胶囊内容为：

```text
‹  286 / 420  ·  68%  ›
```

- 位置编号采用一基计数。
- 百分比优先使用目标 Locator 的 `totalProgression`；缺失时使用 `(target - 1) / (N - 1)` 计算，并限制在 `0...100%`。当 `N == 1` 时显示 `100%`。
- 胶囊固定在底部中央，不跟随手指横向移动。
- 使用系统动态颜色和材质，保证浅色、深色主题下可读。

## 提交与取消

### 提交

- 在有效拖动区域松手时提交当前预览目标。
- 使用 Readium 目标 Locator 直接跳转，不播放普通翻页动画。
- 胶囊在导航完成前保持可见，成功抵达后淡出。
- 只有成功抵达目标后，才允许现有 Reader 流程保存新的 Reading Progress 和同步 Watch 进度。

### 取消

- 手指相对激活点向上移出约 `60 pt` 时进入取消区。
- 取消区内胶囊显示“松开取消”，停止拖动刻度触觉。
- 松手前移回有效区域可恢复目标预览与触觉反馈。
- 在取消区松手时保持原 Reading Position，胶囊淡出。

系统取消手势、设备旋转、分屏尺寸变化、Reader 离开前台或 App 进入后台时，也按取消处理。

## 触觉反馈

- 长按成功：一次轻触觉，确认进入拖动状态。
- 拖动：使用自适应刻度，整段手势最多约 `50` 个触觉刻度。
- 当 `N <= 50` 时，每个 Reading Position 可作为一个刻度。
- 当 `N > 50` 时，将全书范围均匀量化为不超过约 `50` 个刻度；同一刻度内移动不得重复触发。
- 首次抵达开头或结尾时，提供一次比普通刻度更明确的边界反馈；停留在边界时不得重复触发。
- 重新离开边界后再次抵达，可再次提供边界反馈。
- 跳转失败：提供一次错误触觉。

触觉反馈始终作为本交互的一部分启用，并遵循设备能力与系统行为；不提供 App 内开关。

## TTS 协作

- 仅当 TTS 正在播放时，长按成功后暂停 TTS。
- 取消手势或跳转失败后，从原 Reading Position 恢复朗读。
- 成功提交后，从新的 Reading Position 恢复朗读。
- 若进入手势前 TTS 已暂停或停止，结束手势后保持原状态，不自动播放。

## 其他翻页输入

快速位置跳转处于拖动或提交状态时，忽略以下翻页输入且不排队：

- Apple Watch 翻页；
- 音量键翻页；
- 键盘或方向键翻页；
- Reader 其他边缘点击翻页输入。

手势完成、取消或失败后恢复正常接收。

## 失败处理

若 Readium 拒绝目标 Locator 或未能完成导航：

- 保持原 Reading Position；
- 不保存预览目标为 Reading Progress；
- 提供错误触觉；
- 显示短暂提示“无法跳转到该位置”；
- 若进入手势前 TTS 正在播放，则从原位置恢复朗读。

不得自动尝试邻近 Reading Position。

## 状态模型

```text
不可用 ──位置列表准备完成──> 空闲
空闲 ──长按成功──> 预览
预览 ──上移出取消阈值──> 待取消
待取消 ──移回有效区域──> 预览
待取消 ──松手/系统中断──> 空闲（保持原位置）
预览 ──松手──> 提交中
提交中 ──成功──> 空闲（保存新位置）
提交中 ──失败/系统中断──> 空闲（保持原位置）
```

同一时刻只能存在一次快速位置跳转会话。退出任何非空闲状态时，必须清理胶囊、触觉刻度状态、外部输入拦截和临时 TTS 状态。

## 验收标准

1. 在 iPhone 和 iPad 上，EPUB、CBZ、PDF 的分页与滚动模式均可通过长按底部指示器进入快速位置跳转。
2. 短按指示器仍切换导航栏；位置列表未准备好时长按无反应。
3. 长按成功后不移动手指，预览目标等于当前位置，不发生跳变。
4. 从激活点拖到可用范围左边界得到位置 `1`，拖到右边界得到位置 `N`。
5. 拖动时正文和持久化 Reading Progress 不变；只有成功提交后才更新。
6. 胶囊显示正确的目标位置、总位置数和百分比，且在浅色、深色主题中可读。
7. 向上移出取消区后松手不跳转；移回后可以继续预览并提交。
8. 整次拖动触觉刻度不超过约 `50` 个，同一刻度和同一边界不连续重复震动。
9. 松手提交后直接跳转且不播放翻页动画；成功抵达后胶囊淡出。
10. 跳转失败时保留原位置，显示错误提示并提供错误触觉，不尝试邻近位置。
11. TTS 播放中进入手势会暂停；取消后从原处续播，成功后从目标处续播。
12. 拖动和提交期间收到的 Watch、音量键、键盘及其他翻页命令均被忽略且不会延后执行。
13. 旋转、分屏尺寸变化、系统取消、离开前台和进入后台均安全取消会话，不留下胶囊或输入拦截状态。

## 文档决策

本功能属于 Reader 内部的局部、可逆交互，不引入新的架构边界或技术锁定，因此不创建 ADR。
